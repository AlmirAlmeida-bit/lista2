📄 Etapa 3 — O que cada arquivo faz
O projeto é dividido em arquivos com responsabilidades bem separadas. Cada um cuida de uma parte específica da aplicação.
React
App.tsx
É o "porteiro" da aplicação. Verifica se o usuário está logado e decide qual tela mostrar: Login, Cadastro ou Lista de Compras.
React
components/Login.tsx
Tela de login. Tem os campos de e-mail e senha. Quando o usuário clica em Entrar, manda os dados para o Supabase verificar.
React
components/Cadastro.tsx
Tela de criação de conta. Valida se as senhas digitadas são iguais antes de enviar para o Supabase criar o usuário.
React
components/ListaCompras.tsx
Tela principal. Contém todo o sistema de listas: adicionar itens, salvar no banco, editar, excluir e exportar PDF.
Banco
supabaseClient.ts
Arquivo de conexão com o banco de dados. Lê as credenciais do arquivo .env e cria a "ponte" entre o app e o Supabase.
Config
app/.env
Guarda as chaves secretas do Supabase (URL e senha). NUNCA vai para o GitHub — protegido pelo .gitignore.
Config
vite.config.ts
Configura como o projeto é compilado. Define os plugins (React, Tailwind) e o endereço base para o GitHub Pages.
Config
package.json
Lista todas as bibliotecas usadas no projeto. O comando "npm run deploy" faz o build e publica no GitHub Pages automaticamente.
Estilo
src/index.css
Arquivo de estilo global. Importa o Tailwind CSS, que é o sistema de classes de design usado em toda a aplicação.
Config
.gitignore
Lista de arquivos que o GitHub deve ignorar. Protege o .env (credenciais), node_modules (dependências) e dist (build gerado).
🚜 Etapa 4 — Banco de Dados (Supabase)
Os dados das listas de compras são salvos no Supabase em uma tabela chamada listas_compras. Cada usuário só consegue ver e modificar as suas próprias listas — isso é garantido pelas políticas de segurança (RLS).
Coluna Tipo de dado O que guarda
id uuid Identificador único de cada lista — gerado automaticamente pelo banco
user_id uuid ID do usuário dono da lista — vinculado ao sistema de login do Supabase
nome text Nome da lista digitado pelo usuário (ex: "Mercado da semana")
itens jsonb Lista dos itens em formato JSON. Ex: [{ "item": "Arroz", "quantidade": 2, "unidade": "kg" }]
created_at timestamptz Data e hora em que a lista foi criada — preenchido automaticamente pelo banco
🔒 Segurança: Row Level Security (RLS) ativado
Políticas configuradas para as 4 operações: SELECT (ler), INSERT (salvar), UPDATE (atualizar) e DELETE (excluir). Cada usuário logado só tem acesso às listas onde o user_id é igual ao seu próprio ID. Isso significa que nenhum usuário consegue ver ou modificar as listas de outro usuário.
